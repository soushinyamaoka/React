recipe-appのデプロイ作業を実施してください。以下の手順を順番に実行し、各ステップの結果を報告してください。

## 1. 事前チェック

- `git status` で未コミットの変更がないか確認
- 未コミットの変更がある場合は、一覧を表示してユーザーに確認を取る（コミットするか、そのまま進めるか）
- `npx tsc --noEmit` でTypeScriptの型チェックを実行
- エラーがあれば修正を提案し、ユーザーの確認を取る

## 2. 依存関係の確認

- `npx expo install --check` でExpo依存関係の整合性を確認
- 問題があれば `npx expo install --fix` の実行をユーザーに提案

## 3. ビルド実行

- ユーザーにビルド対象プラットフォームを確認する（android / ios / both）
- EAS CLIがインストールされていなければ `npm install -g eas-cli` の実行を提案
- eas.jsonが存在しなければ `eas build:configure` の実行を提案
- 選択されたプラットフォームに対して `eas build --platform <選択>` を実行
- ビルドの進捗URLを表示

## 4. デプロイ確認

- ビルド完了後、`eas build:list` で最新ビルドの状態を確認
- OTAアップデートの場合は `eas update` の使用を提案
- 結果をまとめて報告

## 注意事項

- カスタムAPIサーバーはこのデプロイの対象外です。サーバー側の更新が必要な場合は別途対応してください
- .envファイルやAPIキーがビルドに含まれないことを確認してください
